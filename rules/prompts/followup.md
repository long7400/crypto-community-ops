# Follow-up Prompt

Continue from the current repository state and the active task context.

Before changing code:

1. Briefly restate what is already done.
2. State what remains.
3. Identify the next concrete step.

Then proceed with implementation, validation, and doc updates. If you encounter a decision that belongs to the human, stop and ask only that question.

Use `rules/docs/exec-plans/active/` for active plans and report exact checks run.
