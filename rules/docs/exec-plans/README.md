# Execution Plans

Use this directory for long-running, risky, or cross-cutting work in
`crypto-community-ops`.

- Put active plans in `active/`.
- Move completed plans to `completed/`.
- Keep plans living and current while work proceeds.
- Follow the required sections in `../../../PLANS.md`.
- Name files like `YYYY-MM-DD-short-task-name.md`.

If a plan stops matching reality, update it before continuing.
