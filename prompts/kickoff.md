# Kickoff Prompt

Read `AGENTS.md`, then follow its read order for non-trivial work: `README.md`, `ARCHITECTURE.md`, `PRODUCT_SENSE.md`, `QUALITY_GATES.md`, `TDD_RULES.md`, `SECURITY.md`, `PLANS.md` if planning may be needed, and this task brief: `[path]`.

Then:

1. Summarize the task and the affected subsystems.
2. Identify any ambiguity or risk.
3. Tell me whether this needs a full execution plan or can be implemented directly.
4. If it can be implemented directly, state the first safe implementation step.
5. If it needs a plan, draft it in `docs/exec-plans/active/[filename].md`.

Do not start implementation until you have either determined the task is low risk or asked for approval on the risky parts.
